  • Things to Remember
    • ARIA Labels
      • aria-label: Gives the (non-descriptive) element a descriptive label for the screen reader
      • aria-labelledby: Lets a different element label this element
      • aria-describedby: Points to another element's ID for the description of this element
      • aria-expanded: Says if a menu is expanded
      • aria-hidden: Screen reader will ignore this object
      • aria-invalid: Tells the user if an inputted value is invalid (and potentially why, like grammar, spelling, etc)
      • aria-required: Tells the user this input field is required
      • aria-pressed/checked: Tells the user if this element is pressed or checked
      • aria-disabled: Tells the user if this element is disabled
      • aria-role: Tells the user what kind of role this element fulfils (navigation, button)
      • aria-live: Changes screen reader behaviour; polite waits until user stops interacting to give updates, assertive interrupts
    • Headings
      • h1, h2, etc are important to let screen readers skip to important headings
    • Semantic HTML
      • <header>, <main>, etc are just as important for the same reason
    • Always use alt text for images (or leave empty — screen reader just reads file name)
    • Label every form input, not just placeholders: <label for="inputId">
    • Navigation skip link: <a href=".main">Skip to main</a>
  • Security
    • Never put user inputs directly into SQL, use ? or :named placeholders instead
    • Hash all passwords with DEFAULT_PASSWORD
    • XSS Prevention: htmlspecialchars();

  • PHP Utility 
            Server variables = $_SERVER
        User session variables = $_POST
        Is it a post request? (after form submission)
        $_SERVER['REQUEST_METHOD'] === 'POST'

        Does the key exist?
        isset($_POST["variable"])
        Use coalescing to stop undefined errors:
        Null check: $x = $_POST["variable"] ?? "";
        Falsy check: $x = $_POST["variable"] ?: "";

        Validation:
        filter_var($email, FILTER_VALIDATE_EMAIL);
        filter_var($url, FILTER_VALIDATE_URL);

        Sanitisation:
        filter_var($input, FILTER_SANITIZE_SPECIAL_CHARS)
        filter_var($input, FILTER_SANITIZE_NUMBER_INT)

        Trim whitespace:
        $x = trim($_POST["userInput"])

        String functions:
        strlen($str);
        strtolower($str)/strtoupper($str);
        strcontains($str, "text");
        str_replace("old", "new", $str);

        Array functions:
        count($arr);
        in_array($value, $arr, (opt:type))

        Termination:
        Something wrong: die($value);
        Script stop, nothing wrong: exit($value);

        Date/Time:
        date("Y-m-d") = 2000-12-31 (matches DATE sql column)
        date("y/m/d")
        date("H:i:s") = 23:59:00
        date("Y-m-d H:i:s") = full datetime
        time() = unix timestamp
        strtotime("2000-12-31")
        strtotime("+1 month") = date arithmetic

        Date/Time format letters:
        Y = 2026    = 4 digit year
        y = 26      = 2 digit year
        m = 08      = padded month
        n = 8       = non-padded month
        F = August  = full month name
        M = Aug     = short month name
        d = 01      = padded day
        j = 1       = non-padded month
        l = Monday  = full day name
        D = Mon     = short day name
        H = 23      = 24hr hour
        i = 59      = minute
        s = 00      = second
