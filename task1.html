      • Persona 1 — Jamie (Buyer)
      Age & Occupation 24, University student
      Tech Comfort High — uses smartphone and laptop daily
      Goals Find and purchase niche products quickly; track orders easily
      Pain Points Frustrated by slow checkouts, unclear order status, and hidden fees
      How They Use the Product Browses by category, checks product details before buying, expects a dashboard to track and amend orders
      Accessibility Needs Uses mobile frequently — responsive design and touch-friendly UI are important

    • KPIs
      • Development KPIs
        • At least 80% of planned features finished by deadline
        • 100% of core features implemented by deadline
        • Zero critical bugs at submission
        • Meets accessibility standards like WCAG AA
        • Pages load in under 2 seconds
      • Testing KPIs
        • All pages tested on multiple browsers
        • All forms display appropriate error messages for empty or erroneous inputs
        • User signup, login, etc tested for both account types
        • All access controlled pages reject unauthorised users
        • All SQL inputs tested against injection attempts
        • Responsive design tested on mobile and desktop
