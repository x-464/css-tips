    • Performance
    • Reliability, availability, maintainability and safety
    • Scalability
    • Testability
    • Security
      • CSRF Tokens
      • Use hash_equals() to stop timing attacks related to ===
      • htmlspecialchars() on user inputs/$_GET requests
      • Email validation
      • Password rules
      • Password hashing
      • HTTPS
      • SQL injection prevention by using '?' placeholders in PDO requests
      • Pages protected via user access levels set at account creation
    • Performance:
      • Pages should load within 3 seconds
      • Database queries optimised (e.g., indexed columns, avoiding N+1 queries)
    • Usability:
      • Responsive design across mobile and desktop
      • Users can reach any page within 3 clicks​
      • Clear error messages on form validation
    • Accessibility
      • Semantic HTML
      • ARIA
      • Images (alt tags)
      • Turning off animations for reduced motion
      • Contrasting Colours

      • Test ID Description Type Input / Steps Expected Result Actual Result Pass/Fail
      TC01 Login – normal (valid credentials) Functional Enter correct email and password, click login Redirected to dashboard
      TC02 Login – erroneous (wrong password) Functional Enter correct email, incorrect password, click login Error message displayed, no redirect
      TC03 Login – erroneous (empty fields) Functional Leave email and password blank, click login Validation error shown
      TC04 Password – boundary (minimum length) Functional Register with password exactly at minimum character limit Account created successfully
      TC05 Password – erroneous (below minimum length) Functional Register with password one character below minimum Validation error shown
      TC06 SQL injection attempt on login form Security Enter ' OR '1'='1 into email field Login fails, no DB error exposed

    • KPIs
      • Development KPIs
        • At least 80% of planned features finished by deadline
        • 100% of core features implemented by deadline
        • Zero critical bugs at submission
        • Meets accessibility standards like WCAG AA
        • Pages load in under 2 seconds
      • Testing KPIs
        • All pages tested on multiple browsers
        • All forms validate correctly
        • User signup, login, etc tested for both account types
        • All access controlled pages reject unauthorised users
        • All SQL inputs tested against injection attempts
        • Responsive design tested on mobile and desktop
